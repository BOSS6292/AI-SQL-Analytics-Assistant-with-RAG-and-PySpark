# 🚀 AI SQL Analytics Assistant: RAG-Powered Natural Language Analytics on Azure

Transform natural language questions into SQL queries using Azure OpenAI, retrieve domain-specific query patterns through Azure AI Search Vector Search (RAG), execute queries on large datasets using PySpark, and automatically generate visual insights.

---

## 📌 Project Overview

This project enables users to ask business questions in plain English and receive insights from large datasets without writing SQL manually.

### Example

**User Question**

```text
Show daily rides in July 2024
```

**Generated SQL**

```sql
SELECT
CAST(tpep_pickup_datetime AS DATE) AS ride_date,
COUNT(*) AS total_rides
FROM taxi_trips
GROUP BY CAST(tpep_pickup_datetime AS DATE)
```

**Output**

- SQL generated by GPT-4o
- Query executed using Spark SQL
- Results displayed as charts and tables

# ✨ Key Features

### Natural Language to SQL

Convert business questions into executable SQL queries.

### Retrieval-Augmented Generation (RAG)

Uses Azure AI Search vector indexes to retrieve relevant SQL examples before generation.

### Semantic Search

Embeddings enable retrieval of similar query patterns instead of relying solely on keyword matching.

### Spark SQL Execution

Generated SQL queries are executed on Parquet datasets using PySpark.

### Automated Visualizations

Results are automatically transformed into interactive charts and tables.

### Secure Secret Management

Azure Key Vault is used to securely store:

- Azure OpenAI API Keys
- Azure AI Search Keys
- Service Endpoints

No secrets are stored in source code.

# 🧠 How RAG Works

Instead of relying solely on GPT-4o, the application first retrieves relevant SQL examples from Azure AI Search.

Example vector index:

| Question | SQL |
|-----------|------|
| Average fare amount | SELECT AVG(fare_amount)... |
| Daily ride count | SELECT COUNT(*)... |
| Top pickup locations | SELECT ... |

Workflow:

```text
User Question
       │
       ▼
Embedding Generation
       │
       ▼
Vector Search
       │
       ▼
Retrieve Similar SQL Examples
       │
       ▼
GPT-4o
       │
       ▼
Generate Better SQL
```

This improves SQL accuracy and consistency.

# 📂 Dataset Support

The project currently uses:

```text
yellow_tripdata_2024-07.parquet
```

However, the architecture is designed to support any structured dataset.

To use another dataset:

### Step 1

Replace the Parquet file.

### Step 2

Extract schema using:

```python
df.printSchema()
```

### Step 3

Create Question-SQL examples for the new domain.

Example:

```csv
question,sql
Average salary by department,...
Top 10 employees by salary,...
Monthly sales trend,...
```

### Step 4

Generate embeddings.

### Step 5

Create a new Azure AI Search vector index.

### Step 6

Update:

```python
index_name = "your-new-index"
```

The same architecture can then be used for:

- Sales Analytics
- HR Analytics
- Healthcare Analytics
- Finance Analytics
- E-commerce Analytics
